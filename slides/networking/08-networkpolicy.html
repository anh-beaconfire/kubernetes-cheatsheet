Networking

NetworkPolicy

  • NetworkPolicy defines which Pods can communicate with each other and with external destinations — enforcing zero‑trust networking inside the cluster.
  • Requires a CNI plugin with policy support (Calico, Cilium, Weave Net, Kube‑Router). Policies do nothing without a capable CNI.
  • Start with a default deny policy in sensitive namespaces to block all ingress/egress until explicitly allowed.
  • Explicitly allow only the traffic your application needs — by Pod labels, namespaces, ports, or IP blocks.
  • Test policies carefully using temporary Pods to ensure you don’t accidentally block required traffic.
  • Remember to allow DNS egress (UDP/TCP 53) or your Pods may fail to resolve Service names.
  • NetworkPolicies apply to Pods, not Services — Services inherit the Pod’s allowed/blocked traffic.
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-api-from-frontend
spec:
  podSelector:
    matchLabels:
      app: api
  policyTypes:
    - Ingress
  ingress:
    - from:
        - podSelector:
            matchLabels:
              app: frontend
      ports:
        - protocol: TCP
          port: 8080
