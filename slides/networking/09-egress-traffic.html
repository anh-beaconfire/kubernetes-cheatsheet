Egress Traffic

  • Egress is outbound traffic from Pods to external systems — databases, APIs, SaaS, or the public internet.
  • Common egress targets include:
    • external databases
    • third‑party APIs
    • cloud provider services (S3, Pub/Sub, etc.)
  • Use a static outbound IP (NAT gateway, egress gateway, or cloud NAT) when external systems require IP allowlisting.
  • Firewalls often require egress allowlisting — ensure Pods exit through a predictable IP.
  • Control outbound traffic using NetworkPolicy egress rules to enforce zero‑trust networking.
  • Be mindful of DNS: Pods may need egress to UDP/TCP 53 for name resolution.
  • For advanced control, use an egress gateway (service mesh or CNI feature) to route, inspect, or log outbound traffic.
