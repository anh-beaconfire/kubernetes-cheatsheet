Networking

DNS in Kubernetes

  • Kubernetes automatically creates DNS records for Services (and optionally Pods) using the cluster DNS add‑on (CoreDNS).
  • Always use Service DNS names instead of Pod IPs — Pods are ephemeral, Services are stable.
  • Avoid hardcoding ClusterIPs; rely on DNS for service discovery and load balancing.
  • Each Service gets multiple DNS forms for namespace‑scoped and fully‑qualified lookups.
  • Pod DNS is rarely needed; use it only for advanced debugging or stateful workloads.
  • Debug DNS issues using temporary pods (busybox, alpine) with tools like nslookup, dig, or curl.
  • If DNS fails, check CoreDNS logs, ConfigMap, and network policies that may block DNS traffic (UDP/TCP 53).
