Networking

Gateway API

  • Gateway API is the next‑generation Kubernetes networking API — more expressive, extensible, and role‑oriented than Ingress.
  • GatewayClass — defines the type of data plane (NGINX, Envoy, Traefik, cloud LB); created and owned by platform/infrastructure teams.
  • Gateway — an instance of a GatewayClass; represents the actual load balancer or proxy that receives traffic.
  • HTTPRoute — application‑level routing rules (hosts, paths, headers, filters) that attach to a Gateway.
  • Enables clean separation of responsibilities: infra owners manage Gateways, app teams manage Routes.
  • Supports advanced traffic features: header matching, traffic splitting, retries, timeouts, and filters — without vendor‑specific annotations.
  • Recommended for new platform designs and modern service networking architectures.
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: web-gateway
spec:
  gatewayClassName: nginx
  listeners:
    - name: http
      protocol: HTTP
      port: 80

---

apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: api-route
spec:
  parentRefs:
    - name: web-gateway
  rules:
    - backendRefs:
        - name: api-service
          port: 80
