Networking

Ingress

  • Ingress defines HTTP/HTTPS routing rules that expose internal Services to the outside world.
  • Requires an Ingress Controller (NGINX, Traefik, HAProxy, Envoy, or cloud‑provider controllers).
  • Use TLS to secure traffic; terminate TLS at the Ingress Controller.
  • Configure timeouts, body size limits, rate limits, and headers to protect upstream services.
  • Avoid exposing internal‑only Services; restrict access using path rules, host rules, and NetworkPolicies.
  • Monitor Ingress Controller metrics (latency, 4xx/5xx, saturation) for early detection of issues.
  • Ingress is for HTTP/HTTPS only — use LoadBalancer or NodePort for raw TCP/UDP traffic. 
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: api-ingress
spec:
  ingressClassName: nginx
  rules:
    - host: api.example.com
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: api-service
                port:
                  number: 80
