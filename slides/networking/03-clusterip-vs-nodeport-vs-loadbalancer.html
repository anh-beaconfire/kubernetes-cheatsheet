Networking

ClusterIP vs NodePort vs LoadBalancer

Type Scope Use Case
ClusterIP Accessible only inside the cluster
virtual IP + DNS		 Internal microservices, APIs, and backends
default and most common type
NodePort Exposes a port on every node
nodeIP:nodePort		 Simple external access, dev/test environments
not ideal for production traffic
LoadBalancer Provisions a cloud load balancer
public or private external IP		 Production‑grade external access
recommended for public‑facing services
ExternalName DNS CNAME alias
no proxying, no VIP		 Mapping to external dependencies
databases, SaaS endpoints, legacy systems
Avoid NodePort for production public apps — use LoadBalancer or Ingress instead
