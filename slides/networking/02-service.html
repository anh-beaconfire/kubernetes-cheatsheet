Networking

Service

  • A Service provides stable networking, discovery, and load balancing for a set of Pods.
  • ClusterIP — internal‑only virtual IP; default type; used for service‑to‑service communication inside the cluster.
  • NodePort — exposes the Service on a port on every node; useful for simple external access or when paired with external load balancers.
  • LoadBalancer — provisions a cloud load balancer; exposes the Service externally with a public IP; common for production ingress points.
  • ExternalName — maps a Service to an external DNS name; no proxying, just DNS CNAME resolution.
  • Use ClusterIP for most internal microservices; use LoadBalancer or Ingress for external traffic.
apiVersion: v1
kind: Service
metadata:
  name: api-service
spec:
  type: ClusterIP
  selector:
    app: api
  ports:
    - port: 80
      targetPort: 8080
