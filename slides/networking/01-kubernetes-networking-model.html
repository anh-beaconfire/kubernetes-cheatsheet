Networking

Kubernetes Networking Model

flowchart LR

  %% --- Control Plane ---
  subgraph CP[Control Plane]
    API["API Server
    • Stores Services and Endpoints
    • Provides cluster state"]
  end

  %% --- Worker Node ---
  subgraph NODE[Worker Node]
    INGRESS["1. Ingress or Gateway
    • Receives external HTTP traffic
    • Routes to Services"]

    SERVICE["2. Service
    • Stable VIP and DNS
    • Load balances to Pods"]

    POD["3. Pod
    • Has unique Pod IP
    • Receives traffic directly"]

    CNI["4. CNI Plugin
    • Configures Pod networking
    • IP allocation and routing"]
  end

  %% --- External Client ---
  CLIENT["0. External Client
  • Sends request into cluster"]

  %% --- Connections ---
  CLIENT --> INGRESS
  INGRESS --> SERVICE
  SERVICE --> POD
  POD --> CNI
  API --> SERVICE
