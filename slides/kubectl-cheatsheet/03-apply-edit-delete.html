kubectl

Apply, Edit, Delete

  • Use kubectl apply -f to ensure consistent, repeatable environments across clusters.
  • Preview changes to avoid accidental production modifications.
  • Edit live resources — useful for quick fixes, but changes are not stored in Git unless updated manually.
  • Delete safely — remove resources cleanly using the same manifest or by name.
  • Tip — always understand what the API server will reconcile before applying or deleting.
# Apply declarative config (create or update)
kubectl apply -f app.yaml

# Show what will change before applying
kubectl diff -f app.yaml

# Delete using the same manifest
kubectl delete -f app.yaml

# Edit a live resource (opens editor)
kubectl edit deployment api

# Delete a specific Pod (forces recreation by Deployment)
kubectl delete pod
