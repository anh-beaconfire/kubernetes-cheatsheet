kubectl

Logs, Exec, Port Forward

  • Use logs to understand container behavior, crashes, startup failures, and probe issues.
  • Use exec to inspect the runtime environment: env vars, config files, DNS, network, permissions.
  • Use port-forward to test services locally without exposing them externally.
  • Tip: logs + describe + exec form the core debugging loop for most Pod issues.
# View logs from all Pods in a Deployment
kubectl logs deployment/api

# Stream logs from a specific Pod (live tail)
kubectl logs -f pod/

# View logs from the previous container instance (crash loops)
kubectl logs  --previous

# Exec into a running Pod (debug environment)
kubectl exec -it  -- sh

# Forward local port to a Service (test locally)
kubectl port-forward svc/api-service 8080:80
