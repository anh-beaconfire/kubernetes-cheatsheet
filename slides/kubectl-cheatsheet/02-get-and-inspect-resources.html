kubectl

Get and Inspect Resources

  • List resources quickly — get Pods, Deployments, Services, and Ingress to understand the current state.
  • Use wide output — see Pod IPs, nodes, and readiness for debugging scheduling and networking issues.
  • Inspect resource details — describe shows Events, conditions, container states, and configuration.
  • Understand API structure — explain helps you understand fields, spec structure, and valid configuration.
  • Discover resource types — api-resources shows everything the cluster supports (CRDs included).
  • Senior SWE tip — always start with “get” to understand the surface area, then “describe” to find the root cause.
# 1. List Pods (status, age, restarts)
kubectl get pods

# 2. Wide output (node, IP, readiness)
kubectl get pods -o wide

# 3. List everything in the namespace (quick overview)
kubectl get all

# 4. List specific resource types (can get multiple at once)
kubectl get deploy,svc,ingress

# 5. Inspect a Pod in detail (events, container state, probes)
kubectl describe pod 

# 6. Understand spec fields
kubectl explain deployment.spec

# 7. List all API resources (including CRDs)
kubectl api-resources
