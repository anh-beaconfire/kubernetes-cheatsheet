Context and Namespace Commands

  • A context tells kubectl which cluster, which user credentials, and which default namespace to use.
  • Context = cluster (API server endpoint) + user (credentials / auth) + namespace (optional default)
  • Understand your current context — avoid running commands against the wrong cluster.
  • Switch contexts safely — critical in multi‑cluster or multi‑environment setups.
  • Set default namespace — prevents mistakes and reduces command verbosity.
  • List namespaces — verify environment separation and resource placement.
  • Debug namespace issues — ensure resources exist in the namespace you think they do.
  • Tip — always confirm context before applying manifests or deleting resources.
# Show all contexts (clusters + users + namespaces)
kubectl config get-contexts

# Show current active context
kubectl config current-context

# Switch to a different context (cluster/environment)
kubectl config use-context 

# List all namespaces
kubectl get namespaces

# Set default namespace for current context
kubectl config set-context --current --namespace=

# View full kubeconfig (for debugging)
kubectl config view

# Rename a context (clean up messy kubeconfigs)
kubectl config rename-context old-name new-name

# Delete a context (remove unused clusters)
kubectl config delete-context 

# Check which namespace kubectl will use by default
kubectl config view --minify | grep namespace
