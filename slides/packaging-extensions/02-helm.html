Packaging

Helm

  • Helm is an external tool (not built into Kubernetes) used as the package manager for Kubernetes applications.
  • Uses charts (templates + values) to package reusable applications and deploy them consistently.
  • A Helm chart is a packaged application: templates, default values, metadata, and optional dependencies.
  • Charts are reusable; you typically have one chart per application, not per environment.
  • Environment differences are handled through values files (values-dev.yaml, values-prod.yaml).
  • Commonly used to install third‑party tools (Prometheus, Grafana, NGINX Ingress, cert-manager).
  • Supports parameterized deployments using values.yaml for environment-specific configuration.
  • Developers provide templates + values; Helm renders them into Kubernetes manifests before applying.
# Chart structure
chart/
  Chart.yaml        # metadata (name, version)
  values.yaml       # default configuration
  templates/        # Kubernetes YAML templates
  charts/           # optional dependencies

---
# Add a chart repository
helm repo add bitnami https://charts.bitnami.com/bitnami

# Install the application using the chart
helm install myapp ./myapp-chart

# Upgrade using production values
helm upgrade myapp ./myapp-chart -f values-prod.yaml

# Roll back to a previous version
helm rollback myapp 1

# Uninstall the release
helm uninstall myapp

# Render manifests for debugging (dev environment)
helm template ./myapp-chart -f values-dev.yaml
