Packaging

Kustomize

  • Kustomize is a Kubernetes-native configuration tool built directly into kubectl (no plugins or 3rd-party libraries).
  • Used to customize Kubernetes YAML without templates, keeping manifests declarative and Git-friendly.
  • Ideal for managing dev/staging/prod differences using a base + overlays structure.
  • Common use cases: patching image tags, replicas, env vars, labels, annotations, and environment-specific config.
  • Widely used in GitOps tools (ArgoCD, Flux) and CI/CD pipelines to generate final manifests.
  • Best practice: keep patches small and focused to avoid drift and maintain readability. 
# kustomization.yaml (base)
resources:
  - deployment.yaml
  - service.yaml

patches:
  - path: patch-replicas.yaml

# Apply using kubectl (built-in support)
kubectl apply -k ./overlays/staging

# Build manifests without applying (CI/CD, GitOps)
kubectl kustomize ./overlays/prod
