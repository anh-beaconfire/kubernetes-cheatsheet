CRDs and Operators

  • CRDs extend the Kubernetes API by adding new resource types (Custom Resources).
  • Operators automate domain-specific operational logic using controllers + CRDs.
  • Used to manage complex systems: certificates, monitoring stacks, secrets, databases.
  • Operators run inside the cluster and continuously reconcile desired state.
  • CRDs define the schema; Operators implement the behavior.
  • Examples: cert-manager, Prometheus Operator, External Secrets Operator, database operators.
  • Important: understand operator permissions (RBAC), lifecycle, and failure modes.
