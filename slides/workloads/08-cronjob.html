CronJob

  • A CronJob creates Jobs on a recurring schedule using standard cron syntax
  • Ideal for recurring, time‑based tasks such as nightly cleanup, scheduled reports, periodic syncs, and backups
  • Each execution creates a new Job, which then runs Pods to completion
  • Use concurrencyPolicy to control overlapping runs (Allow, Forbid, Replace)
  • Configure history limits to avoid excessive Job/Pod retention
  • CronJobs should be idempotent — tasks may run more than once if failures occur
  • Use restartPolicy: Never or OnFailure inside the Job template
apiVersion: batch/v1
kind: CronJob
metadata:
  name: cleanup
spec:
  schedule: "0 2 * * *"
  concurrencyPolicy: Forbid
  successfulJobsHistoryLimit: 3
  failedJobsHistoryLimit: 3
  jobTemplate:
    spec:
      template:
        spec:
          restartPolicy: Never
          containers:
            - name: cleanup
              image: busybox
              command: ["sh", "-c", "echo cleanup"]
