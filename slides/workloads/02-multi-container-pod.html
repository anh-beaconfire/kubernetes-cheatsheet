Multi-Container Pod

  • Containers in a Pod share the same network namespace, IP, and volumes
  • Used when containers must run together and cooperate closely
  • Sidecar — adds supporting functionality (logging, proxying, syncing)
  • Adapter — transforms or normalizes output for other systems
  • Ambassador — acts as a local proxy to external services
  • Init Container — runs setup tasks before main containers start
  • Use multi‑container Pods only when lifecycle coupling is required
  • Each Pod still has a single lifecycle — containers start, run, and restart together 
apiVersion: v1
kind: Pod
metadata:
  name: app-with-sidecar
spec:
  containers:
    - name: app
      image: myapp:1.0

    - name: log-sidecar
      image: busybox
      command: ["sh", "-c", "tail -f /logs/app.log"]
      volumeMounts:
        - name: logs
          mountPath: /logs

  volumes:
    - name: logs
      emptyDir: {}
