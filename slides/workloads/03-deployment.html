Deployment

  • A Deployment manages stateless Pods through a ReplicaSet
  • Provides rolling updates, rollbacks, and controlled rollouts
  • Ensures the desired number of replicas are always running
  • Ideal for APIs, web apps, frontend services, and stateless workers
  • Supports surge/availability controls during updates
  • Automatically replaces failed Pods (self‑healing)
  • Use immutable image tags for safe rollouts
  • Not suitable for stateful workloads — use StatefulSet instead
apiVersion: apps/v1
kind: Deployment
metadata:
  name: api
spec:
  replicas: 3
  selector:
    matchLabels:
      app: api

  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxUnavailable: 1
      maxSurge: 1

  template:
    metadata:
      labels:
        app: api
    spec:
      containers:
        - name: api
          image: myorg/api:1.0.0
          ports:
            - containerPort: 8080

---

# Common Deployment commands
kubectl get deployments
kubectl rollout status deployment/api
kubectl rollout history deployment/api
kubectl rollout undo deployment/api
kubectl scale deployment/api --replicas=5
