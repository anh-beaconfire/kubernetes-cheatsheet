DaemonSet

  • A DaemonSet ensures exactly one Pod runs on every selected node (or a subset of nodes)
  • Ideal for node‑level agents that must run everywhere
  • Common use cases: log collectors, monitoring agents, security scanners, CNI components, and node‑level system daemons
  • Automatically deploys Pods to new nodes as they join the cluster
  • Removes Pods from nodes when they are drained or removed
  • Supports node selectors, taints/tolerations, and affinity rules to target specific nodes
  • Not intended for application workloads — use Deployment or StatefulSet instead
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: node-agent
spec:
  selector:
    matchLabels:
      app: node-agent
  template:
    metadata:
      labels:
        app: node-agent
    spec:
      containers:
        - name: agent
          image: busybox
          command: ["sh", "-c", "while true; do echo running; sleep 30; done"]
