Cheat Sheet

Final Summary

Kubernetes manages desired state

The control plane continuously reconciles actual cluster state to match the declarative configuration you apply.

Pods are the execution unit

Containers run inside Pods, which are scheduled onto nodes and represent the smallest deployable compute object.

Deployments manage stateless apps

They provide rolling updates, rollbacks, replica management, and self‑healing for stateless workloads.

Services provide stable networking

They expose Pods via a stable virtual IP and load balance traffic across healthy, ready endpoints.

Ingress/Gateway exposes traffic

They route external HTTP(S) traffic into the cluster using host/path rules and TLS termination.

ConfigMaps & Secrets provide configuration

They inject environment variables, files, and credentials without rebuilding container images.

PVCs provide persistent storage

PersistentVolumeClaims bind to storage backends to support stateful workloads that need durable data.

Probes, resources & autoscaling

Health checks, resource limits, autoscaling, and PDBs ensure reliability, resilience, and production readiness.
