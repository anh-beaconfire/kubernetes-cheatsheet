Storage

Volume Basics

  • Container filesystems are ephemeral — data disappears when a container restarts. Volumes provide persistence or data sharing across containers.
  • emptyDir — temporary storage that lives as long as the Pod; great for caches, scratch space, or shared work directories.
  • configMap — mount configuration files directly into a container; updates require Pod restart unless app reloads dynamically.
  • secret — mount sensitive values as files; Kubernetes keeps them in tmpfs for extra safety.
  • persistentVolumeClaim — request durable storage backed by cloud disks, NFS, or storage systems; survives Pod restarts and rescheduling.
  • projected — combine multiple sources (ConfigMaps, Secrets, ServiceAccount tokens) into a single unified volume.
  • Choose the volume type based on lifecycle needs: temporary, configuration, sensitive data, or long‑term persistence.
# Example: temporary cache shared between containers
volumes:
  - name: cache
    emptyDir: {}

containers:
  - name: api
    volumeMounts:
      - name: cache
        mountPath: /tmp/cache
