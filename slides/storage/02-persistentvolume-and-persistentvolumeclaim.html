Storage

PersistentVolume and PersistentVolumeClaim

  • PersistentVolume (PV) — a cluster‑level storage resource (NFS, cloud disk, SAN, hostPath, etc.).
  • PersistentVolumeClaim (PVC) — a request for storage by a Pod (size, access mode).
  • A PVC binds to a matching PV based on size, access mode, and selectors.
  • Pods mount PVCs — they never mount PVs directly.
  • Access modes:
    • ReadWriteOnce (RWO) — single node read/write
    • ReadWriteMany (RWX) — multiple nodes read/write
    • ReadOnlyMany (ROX) — multiple nodes read‑only
  • Backup stateful data — PVs map to real disks but are not backups.
  • Monitor disk usage and IOPS — storage saturation can cause latency or Pod eviction.
  • Choose reclaim policy: Retain, Delete, or Recycle depending on data lifecycle.
# PersistentVolume (static provisioning)
apiVersion: v1
kind: PersistentVolume
metadata:
  name: api-pv
spec:
  capacity:
    storage: 10Gi
  accessModes:
    - ReadWriteOnce
  persistentVolumeReclaimPolicy: Retain
  hostPath:               # Example only — not for production
    path: /mnt/data

---

# PersistentVolumeClaim that binds to the PV
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: api-data
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 10Gi

---

# Pod mounting the PVC
volumes:
  - name: data
    persistentVolumeClaim:
      claimName: api-data

containers:
  - name: api
    volumeMounts:
      - name: data
        mountPath: /var/lib/api
