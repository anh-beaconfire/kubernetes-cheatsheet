Storage

StorageClass

  • StorageClass defines how Kubernetes dynamically provisions storage (cloud disks, CSI drivers, NFS, etc.).
  • Dynamic provisioning removes the need to manually create PersistentVolumes — faster, safer, and scalable.
  • Use WaitForFirstConsumer so volumes are created in the correct zone after the Pod is scheduled.
  • Choose reclaim policy carefully:
    • Retain for critical data
    • Delete for ephemeral workloads
  • Document StorageClasses so app teams know which one to choose (fast SSD, cheap HDD, encrypted, replicated, etc.).
  • Use annotations inside the StorageClass to describe performance, cost, and intended use.
# Example: documented StorageClass
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: fast-ssd
  # document purposes using annotations / or docs
  annotations:
    storage.kubernetes.io/description: "High-performance SSD storage for latency-sensitive workloads"
    storage.kubernetes.io/cost-tier: "premium"
    storage.kubernetes.io/use-case: "databases, search engines, analytics"
provisioner: kubernetes.io/aws-ebs
parameters:
  type: gp3
  iops: "6000"
  throughput: "250"
reclaimPolicy: Retain
volumeBindingMode: WaitForFirstConsumer
