Troubleshooting

Events and Describe

  • Events show what happened to a Kubernetes object — scheduling, pulling images, restarts, probe failures, etc.
  • kubectl describe gives a detailed view of an object’s state, conditions, and recent Events.
  • Use Events to diagnose scheduling failures, image pull errors, probe failures, and OOMKilled crashes.
  • Check for volume mount errors and node pressure (memory, disk, PID).
  • Look at Conditions, Events, Status, RestartCount, and Last State when debugging Pods.
# Inspect a Pod's full state + Events
kubectl describe pod api-123
# → Check: Conditions, Restarts, OOMKilled, probe failures, image pull issues

--- 

# View all cluster Events in chronological order
kubectl get events --sort-by=.metadata.creationTimestamp
# → Look for: scheduling failures, node pressure, eviction warnings

--- 

# Describe a Deployment (replicas, rollout status, Events)
kubectl describe deployment api
# → Check: unavailable replicas, rollout failures, HPA interactions

--- 

# Inspect node health and pressure conditions
kubectl describe node worker-1
# → Look for: MemoryPressure, DiskPressure, PIDPressure, taints
