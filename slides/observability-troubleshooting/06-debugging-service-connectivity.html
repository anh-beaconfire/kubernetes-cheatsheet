Troubleshooting

Debugging Service Connectivity

  • Check Service definition — verify selector matches Pod labels, correct ports, and correct type (ClusterIP/NodePort/LoadBalancer).
  • Check Endpoints — ensure the Service actually has Pod IPs; empty endpoints means no matching Pods or Pods not Ready.
  • Check Pod readiness — Pods failing readiness probes will not receive traffic.
  • Validate targetPort — ensure the Service’s targetPort matches the container’s actual listening port.
  • Check NetworkPolicies — confirm traffic is allowed between namespaces, Pods, and Services.
  • Check Ingress — verify host/path rules, backend Service name, and TLS configuration.
  • Check DNS — ensure the Service name resolves correctly inside the cluster.
# 1. Check Service definition (ports, selectors)
kubectl get svc
kubectl describe svc api-service

---

# 2. Check Endpoints (should list Pod IPs)
kubectl get endpoints api-service
kubectl get endpointslices api-service

---

# 3. Check Pod labels and readiness
kubectl get pods --show-labels
kubectl describe pod 

---

# 4. Run a debug Pod to test connectivity
kubectl run debug --image=busybox -it --rm -- sh

---

# 5. Test DNS and HTTP connectivity from inside cluster
nslookup api-service
wget -qO- http://api-service
Error Seen Common Reason Common Fix
No endpoints available for service Service selector does not match any Pod labels Fix Service selector; verify Pod labels with kubectl get pods --show-labels
Endpoints: Pods exist but are not Ready (failing readiness probe) Fix readiness probe, check container logs, ensure app is listening
Connection refused Service targetPort does not match containerPort; app not listening Correct targetPort; verify containerPort via kubectl describe pod
NetworkPolicy denied Ingress/Egress rules block traffic between Pods or namespaces Update NetworkPolicy to allow required Pod/namespace/port traffic
Ingress: backend service not found Ingress rule points to wrong Service name or port Fix Ingress backend serviceName/servicePort
Ingress: no matching host/path Request does not match any Ingress rule Correct host/path rules; verify with kubectl describe ingress
DNS lookup failed Service name incorrect or wrong namespace; CoreDNS issue Use nslookup; check FQDN (svc.namespace.svc.cluster.local)
I/O timeout Pod unreachable due to NetworkPolicy, node issue, or wrong port Test connectivity from debug Pod; verify policies and ports
