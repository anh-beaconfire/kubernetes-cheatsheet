How does a request from the internet reach a pod?

  • Client sends HTTP(S) request to a domain on the public internet.
  • DNS resolves the domain to a public IP address of a cloud load balancer.
  • The cloud load balancer forwards the request to a Kubernetes node running the ingress controller.
  • Traffic reaches the ingress controller Service (typically type=LoadBalancer or NodePort).
  • The ingress controller evaluates Ingress/Gateway rules (host, path, TLS) to select a backend Service.
  • The selected Kubernetes Service (usually ClusterIP) load balances the request across matching Pods.
  • kube-proxy (iptables/IPVS) routes the request from the node to the chosen Pod IP and target port.
  • The Pod's container receives the request on its container port and processes it.
