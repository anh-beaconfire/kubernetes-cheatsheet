Security

SecurityContext

  • SecurityContext defines Linux security settings for Pods and containers (UID/GID, capabilities, seccomp, filesystem rules).
  • Configured in the Pod spec or per‑container container.securityContext inside Deployments, StatefulSets, DaemonSets, Jobs, etc.
  • Targets process privileges, filesystem access, and kernel attack surface for each container.
  • Run workloads as non‑root to reduce blast radius and prevent privilege misuse.
  • Disable privilege escalation to block setuid binaries and container breakout attempts.
  • Drop all Linux capabilities unless explicitly required.
  • Use a read‑only root filesystem to prevent tampering and persistence.
  • Avoid privileged Pods — they bypass container isolation and expose the host.
  • Use seccomp and AppArmor to restrict system calls and reduce kernel attack surface.
# Pod-level security context
securityContext:
  runAsNonRoot: true
  runAsUser: 10001
  seccompProfile:
    type: RuntimeDefault

containers:
  - name: api
    image: myorg/api:1.0.0
    securityContext:
      allowPrivilegeEscalation: false
      readOnlyRootFilesystem: true
      capabilities:
        drop: ["ALL"]
