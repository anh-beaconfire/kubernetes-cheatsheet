Security

Kubernetes Security Layers

  • User / CI — entry point; all requests go through the API server.
  • Authentication — verifies identity using certificates, tokens, or OIDC.
  • RBAC Authorization — checks whether the authenticated identity is allowed to perform the requested action.
  • Admission Control — validates or mutates incoming requests (OPA Gatekeeper, Kyverno, ResourceQuota, LimitRanger).
  • Pod Security (PSA) — enforces Pod-level security standards (privileged / baseline / restricted).
  • Runtime Security — applies seccomp, AppArmor, SELinux, sandboxing, and runtime monitoring.
  • NetworkPolicy — restricts Pod-to-Pod and Pod-to-external communication.
