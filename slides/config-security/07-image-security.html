Security

Image Security

  • Use trusted base images (official, vendor‑maintained, or internally curated).
  • Remove package managers (npm, mvn, pip3) and build tools from runtime images.
  • Use non‑root users inside the image to reduce privilege risks.
  • Use multi‑stage builds to separate build dependencies from runtime.
  • Enable seccomp/AppArmor defaults when running the container.
  • Minimize image size to reduce attack surface and speed up deployments.
  • Avoid the latest tag — it breaks reproducibility and rollbacks.
  • Pin image versions using tags or digests to ensure deterministic builds.
  • Scan images in CI/CD for vulnerabilities before pushing to registries.
  • Store images in private registries with access control and audit logs.
# Multi-stage Dockerfile with secure practices
# Java Spring boot
# 1. Build stage (full JDK + Maven)
FROM maven:3.9-eclipse-temurin-21 AS builder
WORKDIR /app
COPY pom.xml .
RUN mvn -q dependency:go-offline
COPY src ./src
RUN mvn -q package -DskipTests

# 2. Runtime stage (minimal JRE only)
FROM eclipse-temurin:21-jre
WORKDIR /app

# Create non-root user
RUN useradd -u 10001 appuser

# Copy only the final JAR
COPY --from=builder /app/target/*.jar app.jar

USER appuser
ENTRYPOINT ["java", "-jar", "app.jar"]
