Object Basics

Labels and Selectors

  • Labels are key‑value metadata added under metadata.labels
  • Selectors appear in objects that target other resources (e.g., Services, Deployments)
  • Services use selectors to route traffic to matching Pods
  • Deployments use selectors to manage their Pods
  • Monitoring and logging systems group workloads using labels
  • Policies (NetworkPolicy, PodSecurity) match workloads via labels
  • Use consistent labeling conventions across your cluster
# Labels live inside metadata.labels
metadata:
  labels:
    app: payment-api
    tier: backend
    env: prod

---

# Selectors appear in objects that target Pods
selector:
  matchLabels:
    app: payment-api
