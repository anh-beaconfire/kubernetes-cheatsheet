Object Basics

Namespaces

  • A namespace is a logical partition inside a cluster
  • Used to separate teams, applications, or environments
  • Ideal for isolating dev / staging / prod
  • Allows applying resource quotas and network policies
  • Improves organization and access control (RBAC)
  • Not all resources are namespaced (e.g., Nodes, PVs)
  • Avoid putting everything in the default namespace
# A namespace object
apiVersion: v1
kind: Namespace
metadata:
  name: dev

---

# Common namespace commands
kubectl get namespaces
kubectl create namespace dev
kubectl get pods -n dev
kubectl config set-context --current --namespace=dev
