Object Basics

Kubernetes Object Anatomy

  • apiVersion — API group and version for this object
  • kind — the type of Kubernetes resource
  • metadata — name, labels, annotations, namespace
  • spec — desired state defined by the user
  • status — actual state reported by the cluster
  • Kubernetes controllers work to make status match spec 
apiVersion: apps/v1
kind: Deployment
metadata:
  name: api
  labels:
    app: api
spec:
  replicas: 3
  selector:
    matchLabels:
      app: api
  template:
    metadata:
      labels:
        app: api
    spec:
      containers:
        - name: api
          image: nginx:1.27
