Application Deployment Best Practices

Use Deployments for stateless apps

Ideal for horizontally scalable services where Pods can be replaced freely without preserving identity.

Use StatefulSets for stable identity

Required when Pods need persistent identity, stable network names, or attached storage (databases, queues).

Set requests and limits

Guarantees minimum resources and prevents noisy-neighbor issues; enables proper scheduling and autoscaling.

Use readiness/liveness/startup probes

Ensures Pods only receive traffic when ready, restarts unhealthy containers, and protects slow-start apps.

Use ConfigMaps and Secrets

Store configuration outside the container image; separate code from environment-specific settings.

Use immutable image tags

Pin images by version or digest to ensure deterministic rollouts and reliable rollbacks.

Use rolling updates

Gradually replace Pods to avoid downtime; tune maxSurge and maxUnavailable for safe production rollouts.

Configure PDBs

Define how many Pods must remain available during disruptions to avoid outages during node drains or rollouts.
