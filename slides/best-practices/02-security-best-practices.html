Security Best Practices

Least privilege RBAC

Grant only the permissions a workload or user needs; reduce blast radius and limit lateral movement.

Avoid cluster-admin

Cluster-wide admin rights bypass all controls; use scoped roles and service accounts instead.

Run containers as non-root

Minimizes impact of container escape or compromised processes; enforce with securityContext.

Drop capabilities

Remove unnecessary Linux capabilities to shrink the attack surface; follow a minimal privilege model.

Restrict egress

Limit outbound traffic to prevent data exfiltration and enforce zero-trust boundaries between services.

Use NetworkPolicies

Define which Pods can communicate; isolate workloads and block unintended east-west traffic.

Scan images

Detect vulnerabilities early using CI/CD scanners; ensure base images and dependencies are patched.

Do not commit secrets

Store secrets in Secret managers or encrypted stores; enforce scanning to prevent accidental leaks.
