Architecture

Control Plane Components

Component Responsibility
API Server Central management interface that exposes the Kubernetes API, validates requests, and updates cluster state in etcd
etcd Distributed key‑value store that persists all cluster data — configuration, secrets, and resource definitions
Scheduler Assigns Pods to nodes based on resource availability, affinity rules, and scheduling policies
Controller Manager Runs control loops that monitor cluster state and take actions to achieve the desired state (e.g., creating Pods, updating replicas)
Cloud Controller Manager Integrates Kubernetes with cloud provider APIs to manage load balancers, volumes, and node lifecycle events
The control plane maintains cluster state and makes scheduling and orchestration decisions, while worker nodes execute workloads.
